# **CCNA学習中に出てきた用語集**

## **FastEthernet**

```Text
Fast Ethernet（ファストイーサネット）は、コンピュータネットワークにおける通信技術の1つであり、オリジナルのイーサネットよりも高速なデータ転送を提供します。イーサネットはLAN（Local Area Network）で広く使用される規格で、コンピュータやネットワークデバイスを相互に接続し、データの送受信を行うための方法を定義しています。

オリジナルのイーサネットの速度は10 Mbps（メガビット毎秒）でしたが、Fast Ethernetでは100 Mbpsに向上しました。つまり、Fast Ethernetはオリジナルのイーサネットの10倍の速度でデータを送受信できます。

Fast Ethernetは1990年代初頭に登場し、その高速なデータ転送性能と互換性のあるハードウェアの採用により、従来のイーサネットとの移行がスムーズに行われました。現在では、Fast Ethernetはさらなる高速化が求められる場面では過去の技術として扱われることがありますが、一般的なオフィスや家庭のLANなどで広く使用されています。また、より高速なイーサネット規格であるGigabit Ethernet（ギガビットイーサネット）や10 Gigabit Ethernet（10ギガビットイーサネット）が登場しており、データ通信の要件に応じて適切なイーサネット規格を選択することが重要です。
```

## **192.168.1.0/24って**

```Text
192.168.1.0/24は、IPアドレスのサブネット表記法の一つで、プライベートIPアドレスの範囲を示すものです。

IPアドレスはネットワーク上のコンピューターやデバイスを一意に識別するための番号です。IPv4では、IPアドレスは32ビットのビット列で表現され、通常は4つの8ビットのブロック（オクテット）に分割されてドットで区切られた形式で表記されます（例：192.168.1.0）。

サブネットマスクは、IPアドレスのネットワーク部分とホスト部分を区別するために使用されます。CIDR（Classless Inter-Domain Routing）表記法を用いて、IPアドレスのネットワーク部分を示すために「/」の後にビット数を指定します。たとえば、192.168.1.0/24は、IPアドレスの先頭24ビットがネットワーク部分であり、残りの8ビットがホスト部分を表しています。

192.168.1.0/24の場合、ネットワークアドレスは192.168.1.0であり、有効なIPアドレス範囲は192.168.1.1から192.168.1.254までの254個のアドレスです。そして、最後のアドレス（192.168.1.255）はブロードキャストアドレスとして予約されています。

このようにサブネットマスクを指定することで、IPアドレスの範囲を区別し、ネットワークを効率的に管理することが可能となります。
```

## **クロスオーバーケーブルとは**

```Text
クロスオーバーケーブル（Cross-over cable）は、コンピュータネットワークにおいて、同じ種類のネットワーク機器（例：コンピュータとコンピュータ、スイッチとスイッチ）を直接接続するための特別なイーサネットケーブルのことを指します。

通常のイーサネットケーブルは、1つの機器（例：コンピュータ）とネットワークスイッチやハブといったネットワークインフラストラクチャを接続するために使用されます。これはストレートスルーケーブルと呼ばれるタイプのケーブルで、配線が両端で同じ順序になっています。

しかし、2つの同じ種類のネットワーク機器を直接接続する場合（例：コンピュータとコンピュータ、スイッチとスイッチ）、通常のストレートスルーケーブルではうまく通信できないことがあります。これは、両端の機器が同じ送受信のペアで通信するため、データがうまく相互に送信されないためです。

クロスオーバーケーブルは、この問題を解決するために使用されます。このケーブルは、片端ともう片端で配線の順序がクロスオーバーされているため、2つの同じ種類の機器を直接接続する場合に、正しく通信が行われるようになります。これにより、コンピュータ同士の直接接続やスイッチ同士の直接接続が可能になります。

ただし、現代のネットワーク機器の多くはオート・ミディア・センス（Auto MDI-X）と呼ばれる機能をサポートしており、クロスオーバーケーブルを使わずとも、通常のストレートスルーケーブルで直接接続することができる場合もあります。しかし、クロスオーバーケーブルは特定のシチュエーションで依然として役立つことがあります。
```

## **Ethernet0/0とEthernet0/1**

```Text
Ethernet0/0とEthernet0/1をつなぐのは、ネットワーク機器（通常はルーターやスイッチ）のインターフェース間でトラフィックを転送するためです。

ネットワーク機器には複数のインターフェースがあり、それぞれが異なるネットワークセグメントに接続されることがあります。例えば、Ethernet0/0はネットワークAに、Ethernet0/1はネットワークBに接続されている場合を考えてみましょう。

これらのインターフェースをつなぐことで、ネットワークAとネットワークBの間でデータの転送や通信が可能になります。ルーターの場合、Ethernet0/0がWAN側（外部ネットワークやインターネットと接続する側）、Ethernet0/1がLAN側（内部ネットワークと接続する側）に接続されることが一般的です。

このように、Ethernet0/0とEthernet0/1をつなぐことで、ネットワーク機器は異なるネットワーク間でデータをルーティングしたり、スイッチングしたりすることができます。例えば、インターネットからのリクエストを受け取ったルーターは、それをLAN側に接続されたコンピュータに転送して応答を得るなどの役割を果たします。

このようなネットワークの構成により、複数のネットワークを連結し、データの通信や共有を可能にすることができます。

Ethernet0/1は、ネットワーク機器（通常はルーターやスイッチ）のインターフェースの1つを指します。

ネットワーク機器は、通常複数のインターフェース（物理的な接続ポート）を持ち、各インターフェースは特定のネットワークに接続されます。インターフェースは通常「0/0」、「0/1」、「1/0」のように番号付けられ、これによりどのインターフェースがどのネットワークに接続されているかを識別します。

例えば、ルーターの場合、Ethernet0/0はWAN（Wide Area Network）側のインターフェースであり、インターネットサービスプロバイダ（ISP）と接続するためのポートとして使用されることがあります。一方、Ethernet0/1はLAN（Local Area Network）側のインターフェースであり、内部のネットワークに接続されたデバイス（コンピュータや他のネットワーク機器）と通信するためのポートとして使用されることがあります。

これらのインターフェースは、異なるネットワークセグメントとの接続やデータ転送を行うために使用されます。ネットワーク機器の設定や管理では、これらのインターフェースに対して適切な設定を行うことが重要です。また、これらのインターフェースを適切に接続することで、ネットワークの正常な動作を確保することができます。

```

## **WANとLAN**

```Text
Wide Area Network（広域ネットワーク、WAN）は、地理的に広い範囲に広がる複数の地域や場所を接続するコンピュータネットワークのことを指します。

WANは、ローカルエリアネットワーク（Local Area Network、LAN）やメトロエリアネットワーク（Metropolitan Area Network、MAN）とは異なり、広範な地域をカバーするため、より大きなスケールでの通信を可能にします。LANは通常、同じ建物やキャンパス内のコンピュータやデバイスを接続するために使用されるのに対して、WANは地理的に離れた場所にある異なるオフィス、支店、データセンター、または国や大陸に分散した複数の拠点を接続するために利用されます。

WANは、通信事業者のインフラストラクチャやインターネットを利用して構築されることが一般的です。一般的なWANの接続方式には、専用の通信回線（例：専用線、フレームリレー、ATM）、インターネットVPN（Virtual Private Network）、MPLS（Multiprotocol Label Switching）などがあります。

WANの利点は、地理的に離れた場所のユーザーやリソースを一体化し、データの共有やアプリケーションの利用を効率的に行うことができることです。また、複数の拠点を接続することで、情報のセキュアな共有やビジネスコラボレーションを強化することも可能です。
```


## **ストレートスルーケーブル**

```Text
コンピュータやネットワーク機器を接続するためのケーブルの一種です。このタイプのケーブルは、特定の機能や変換をせずに、単に接続したいデバイス同士を直接接続するために使用されます。ここで「スルー（Through）」は、信号が何らかの変換や処理を経ずに直接通過することを意味します。

ストレートスローケーブルは、通常は同じタイプのコネクタ（例: イーサネットケーブルのRJ45コネクタ）を持つデバイスを接続する際に使用されます。例えば、コンピュータとスイッチ、コンピュータとルータ、またはコンピュータ同士を接続する場合などです。この場合、ケーブルの両端に同じピン配置が必要です。

一方で、異なる機能や規格を持つデバイスを接続するためには、クロスオーバーケーブルと呼ばれる特別なケーブルが必要です。クロスオーバーケーブルは、通信の送受信ラインが交差しているため、一方のデバイスの送信信号が他方のデバイスの受信信号に接続されるようになっています。

これらのケーブルの使い分けは、デバイス間の通信がどのように行われるかに依存します。一般的な利用ケースに合わせて適切なケーブルを選ぶことが重要です。
```

## **VLANとは**

```Text
VLAN（Virtual LAN）は、ネットワークのセグメンテーションや論理的な分割を行うための技術です。VLANを使用することで、物理的なネットワークインフラストラクチャに関わらず、論理的なネットワークセグメントを作成し、異なるネットワークデバイスを同じ物理的なネットワーク上に隔離して配置できます。

VLANを導入することで、以下のような利点があります：

セキュリティとアイソレーション: VLANを使用することで、異なるセグメントのデバイス同士を隔離できます。これにより、セキュリティ上のリスクを軽減し、ネットワークの一部に障害が発生した場合でも他の部分に影響を与えることが少なくなります。

トラフィック管理: VLANを使用することで、ネットワークトラフィックを論理的に分割することができます。これにより、トラフィックの優先度を設定したり、ネットワークリソースを効果的に管理したりできます。

論理的なネットワーク構築: 物理的な制約に縛られずに論理的なネットワークセグメントを構築できるため、柔軟なネットワークアーキテクチャを実現できます。例えば、異なる部署やプロジェクトが異なるVLAN内で作業することができます。

効率的なリソース利用: 物理的なネットワーク機器をより効果的に活用することができます。特定のタイプのトラフィックが特定のVLANに制限されるため、ネットワーク帯域幅を最適化することができます。

VLANはスイッチやルータなどのネットワーク機器で設定され、各VLANは論理的に分割されたセグメントとして機能します。通常、異なるVLAN間の通信が必要な場合は、ルータやレイヤ3スイッチを介して通信を行います。
```

## **サブインターフェイスとは**

```Text
サブインターフェイスは、コンピュータネットワークや情報技術における用語で、主にネットワーキングデバイス（ルーターやスイッチなど）に関連して使用されます。インターフェイスは、デバイスがネットワーク上で通信を行うための接続点を指します。サブインターフェイスは、これらの主なインターフェイスの下位に作成される論理的なインターフェイスです。

一般的に、サブインターフェイスは同じ物理的なインターフェイス（通常はネットワークポート）を複数の論理的なセグメントに分割するために使用されます。これにより、1つの物理インターフェイスを使用して、異なるVLAN（仮想LAN）や異なるネットワークセグメントにトラフィックを分割することができます。

サブインターフェイスは通常、VLANタギングプロトコル（例：802.1Q）を使用して、異なるVLANのトラフィックを区別します。また、異なるIPアドレス範囲やサブネットを持つ論理的なセグメントを作成するためにも使用されることがあります。

例えば、1つの物理インターフェイスを使用して、ルータが複数の顧客の異なるVLANに対応する場合、それぞれのVLAN用にサブインターフェイスを設定することができます。これにより、異なるVLAN間での通信が論理的に分離され、管理が容易になります。

要するに、サブインターフェイスは、1つの物理インターフェイスを論理的に分割し、異なるネットワークセグメントやVLAN間でトラフィックを制御するために使用される仕組みです。
```

## **Router on a stickとは**

```Text
「Router On A Stick」（ルーター・オン・ア・スティック）は、コンピュータネットワークにおける設定の一種で、1つの物理的なインターフェースを使用して複数の仮想インターフェース（サブインターフェース）を作成し、異なるVLAN間の通信をルーティングするための手法です。

通常、ルーターは異なるサブネット間の通信を処理するために使用されますが、ルーターが複数のVLANに接続されている場合、物理インターフェースが限られていることがあります。そこで、「Router On A Stick」では、1つの物理インターフェースを複数の論理的なインターフェースに分割し、各VLAN間の通信を処理する仕組みを構築します。

具体的な手順は以下の通りです：

1つの物理インターフェースを設定します。通常、802.1Qタギングプロトコルを使用してVLANタグが付与されたトラフィックを受信するために使用されます。

設定された物理インターフェース上に複数のサブインターフェースを作成します。各サブインターフェースは異なるVLANに関連付けられます。これにより、1つの物理インターフェースを複数の論理的なルーターインターフェースとして使用できます。

各サブインターフェースに対して適切なIPアドレスやルーティング設定を行います。

ルーターが異なるVLAN間の通信をルーティングするための設定を行います。これにより、サブインターフェース間でトラフィックがルーティングされ、異なるVLAN間で通信が可能になります。

「Router On A Stick」は、特に物理インターフェースが限られている状況や、コスト効率を重視する場合に有用です。ただし、大量のトラフィックが発生する場合や、高い性能が求められる場合には適切でないこともあります。
```

## **Access Control List(ACL)とは**

```Text
アクセス制御リスト（Access Control List、ACL）は、コンピュータネットワークやオペレーティングシステムにおいて、リソースへのアクセス権限を管理するための仕組みです。ACLは、特定のユーザーやグループがファイル、フォルダ、ネットワークリソースなどに対してどのような操作を行えるかを制御します。

ACLは通常、以下の要素で構成されています：

対象リソース（Resource）: アクセス権限を設定したいファイル、ディレクトリ、ネットワークリソースなどの対象物を指します。

対象ユーザーまたはグループ（Subject）: アクセス権限を付与する対象のユーザーやグループを指します。これにより、特定のユーザーやグループがアクセス権を持つかどうかが制御されます。

アクセス権限（Permissions）: 対象ユーザーやグループに許可される操作やアクションを示すもので、読み取り、書き込み、実行などが含まれます。一般的なアクセス権限には「読み取り専用」、「読み取りと書き込み」、「フルアクセス」などがあります。

アクセス制御リストは、セキュリティの向上やプライバシーの保護などの目的で使用されます。例えば、特定のファイルを一部のユーザーだけが読み取ることができるように制御したり、特定のグループだけが特定のディレクトリに書き込み操作を行えるようにしたりするために使用されます。

オペレーティングシステム、ネットワーク機器、データベースシステムなど多くのシステムでACLが利用されており、セキュリティとアクセス制御の重要な機能を担っています。
```
